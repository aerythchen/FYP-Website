The problem

IoT deployments stream frequent measurements from constrained devices. Traditional per‑message signatures bloat payloads and verification cost. Managing certificates is complex and ill‑suited to ad‑hoc device identity like MAC addresses or serial numbers.

  • High bandwidth overhead from individual signatures
  • Expensive verification at gateways/cloud
  • Certificate lifecycle complexity

The solution: IBAS

Gentry’s Identity‑Based Aggregate Signatures allow multiple signers to produce signatures that can be aggregated into a single compact value. In identity‑based crypto, a signer’s identifier (e.g., device MAC or serial) replaces certificates, simplifying deployment.

  • Aggregate many signatures into one
  • Verify once for all contributors
  • Use device identifiers directly

How it works

1. Setup

A Private Key Generator (PKG) initializes public parameters and issues device private keys for identifiers.

2. Sign

Each device signs its weather measurement with its identity‑based key.

3. Aggregate

A gateway aggregates signatures from many devices into one compact signature.

4. Verify

The server verifies a single aggregate signature against all identities and messages.
